# 전체 모노레포 커밋 규칙 (Mono-repo Commit Convention)
앞으로 여러 가지 자동화 및 유틸리티 프로젝트가 이 최상위 폴더 밑으로 모이게 됩니다. AI(에이전트)와의 티키타카로 이루어지는 **'바이브 코딩'** 환경에서, 과거에 뭘 만들었고 어딜 고쳤는지 한눈에 파악하기 위해 아래의 커밋 규칙을 전역적으로 적용합니다.
## 📝 커밋 메시지 포맷
[폴더명] 타입: 커밋 내용 기재
## 📌 타입(Type) 분류 가이드
1. Init : 새로운 유틸리티 폴더 세팅 (뼈대 구축)
2. Feat : 새로운 기능, UI 패널 추가
3. Fix : 버그, 에러 로직, 뷰 깨짐 해결
4. Docs : AGENTS.md, PROGRESS.md 등 기획서 문서화
5. Style : 픽셀, CSS 다크테마 등 UI 디자인 변경
