# 전체 모노레포 커밋 규칙 (Mono-repo Commit Convention)

앞으로 여러 가지 자동화 및 유틸리티 프로젝트가 이 최상위 폴더 밑으로 모이게 됩니다. AI(에이전트)와의 티키타카로 이루어지는 **'바이브 코딩'** 환경에서, 과거에 뭘 만들었고 어딜 고쳤는지 한눈에 파악하기 위해 아래의 커밋 규칙을 전역적으로 적용합니다.

## 📝 커밋 메시지 포맷
```text
[폴더명] 타입: 커밋 내용 기재 (필요시 상세 내용)
```

## 📌 타입(Type) 분류 가이드
1. **Init** : 새로운 유틸리티 폴더를 처음 파고 세팅할 때 (뼈대 구축)
2. **Feat** : 새로운 기능, UI 패널, 모드 등을 추가할 때
3. **Fix** : 버그, 에러 로직, 화면 깨짐 등을 고칠 때
4. **Docs** : 그 폴더의 `AGENTS.md`, `PROGRESS.md`, `README.md` 등 기획서나 문서를 업데이트할 때
5. **Style** : 기능 로직 변화 없이 색상(다크테마), 픽셀 등 UI 디자인이나 코드 들여쓰기만 변경할 때
6. **Refactor** : 기능이나 동작은 똑같으나 안의 코드 구조만 깔끔하게 청소할 때

## 💡 스크립트 작성 (바이브 코딩) 예시
- `[image_analyzer] Feat: v1.4 통계 패널 UI 및 OpenCV 수치 연산 추가`
- `[image_analyzer] Fix: 히스토그램 캔버스 1픽셀 간격 UI 깨짐 수정`
- `[pdf_merger] Init: 신규 PDF 병합기 유틸리티 뼈대 폴더 생성`
- `[공통] Docs: 전체 모노레포 커밋 규칙 세팅 및 초기화`
